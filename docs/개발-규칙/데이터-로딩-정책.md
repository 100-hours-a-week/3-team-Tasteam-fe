# 데이터 로딩 정책

이 문서는 실제 서비스 기준의 데이터 로딩 규칙을 정의합니다. 개발 편의를 위한 더미 데이터는 사용하지 않습니다.

## 1) 더미 데이터 금지

- 앞으로 **더미 데이터는 전부 제거**한다.
- 개발 중에도 UI 확인이 필요하면 **실제 API** 또는 **공용 기본값(디폴트 값)**으로만 확인한다.

## 2) 데이터 로딩 전 기본 상태

페이지 첫 렌더링 시점의 기본 상태는 다음과 같다.

- 컨텐츠가 없는 **빈 상태**가 기본이다.
- 단, **레이아웃 골조는 유지**되어야 한다.

## 3) API 결과에 따른 표시 규칙

- **API 호출 성공** → 실제 데이터 표시
- **API 호출 실패** → 기존에 합의된 **기본 디폴트 값** 또는 **데이터 없음** 상태 표시
  - 기본 이미지/기본 문자열/빈 문자열 등 **문맥에 맞는 기본값**을 선택한다.

## 4) CLS 방지 (스켈레톤 UI 필수)

- CLS(레이아웃 쉬프트)를 방지하기 위해 **스켈레톤 UI**를 반드시 작성한다.
- 데이터가 들어오기 전에는 **기본 골조만 존재**하고, 컨텐츠는 비어 있어야 한다.
- 데이터가 들어오면 **동일한 레이아웃 위에서 실제 값만 치환**되어야 한다.

## 체크리스트

- 더미 데이터가 남아 있지 않은가?
- 로딩 전에도 레이아웃 골조가 유지되는가?
- 실패 시 기본값/데이터 없음 상태가 명확한가?
- 스켈레톤 UI로 CLS가 방지되는가?
