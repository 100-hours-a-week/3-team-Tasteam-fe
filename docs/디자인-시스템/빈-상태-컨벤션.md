# 빈 상태(검색 결과 없음) UI 컨벤션

이 문서는 **검색 결과가 없을 때 보여주는 UI**를 일관되게 유지하기 위한 규칙을 정리합니다.
에러(오류) 상태와 혼동되지 않도록 **중립적인 톤**을 유지하는 것이 핵심입니다.

## 1) 목적

- 사용자에게 "오류"가 아닌 "결과 없음"을 명확히 전달
- 화면 간 UI 톤과 구성 요소(아이콘/문구/버튼)를 통일
- 최소한의 인지 부담으로 다음 행동(검색어 변경, 그룹 만들기 등) 유도

## 2) 톤 & 룩(필수)

- **중립색** 사용: `muted` 계열 사용
- **경고/에러 색상 금지**: `destructive` 컬러 및 경고 아이콘 사용 금지
- 아이콘은 **검색/빈 상태**를 연상시키는 중립 아이콘 사용
  - 기본: `Search` (lucide-react)

## 3) 레이아웃 규칙

- 중앙 정렬(상단 여백 충분히 확보)
- 아이콘 → 타이틀 → 설명 → 버튼 순서
- 텍스트는 짧고 단순하게 유지

권장 구조(예시):

```tsx
<Container className="py-12">
  <div className="flex flex-col items-center text-center">
    <div className="w-14 h-14 rounded-full bg-muted flex items-center justify-center mb-4">
      <Search className="w-6 h-6 text-muted-foreground" />
    </div>
    <h3 className="text-base font-semibold mb-2">검색 결과가 없습니다</h3>
    <p className="text-sm text-muted-foreground">다른 키워드로 다시 검색해보세요.</p>
  </div>
</Container>
```

## 4) 문구 규칙

기본 문구(권장):

- 타이틀: **"검색 결과가 없습니다"**
- 설명: **"다른 키워드로 다시 검색해보세요."**

카테고리별 빈 상태 문구(필요 시):

- 그룹 없음: **"그룹 검색 결과가 없습니다"**
- 음식점 없음: **"음식점 검색 결과가 없습니다"**

## 5) 버튼 규칙

버튼은 "추가 행동이 명확할 때만" 노출합니다.

- 그룹만 없는 경우 → **"그룹 만들기"** 버튼 노출
- 둘 다 없는 경우 → 기본적으로 버튼 없이 안내만 표시

버튼 스타일:

```tsx
<Button variant="outline" size="sm" className="border border-muted-foreground/50">
  그룹 만들기
</Button>
```

## 6) 에러 상태와 구분

빈 상태는 **에러 페이지와 동일한 톤을 사용하지 않습니다.**

- 경고 아이콘 금지 (예: AlertTriangle)
- 붉은 색상 금지 (`destructive`)
- "오류", "문제 발생" 문구 금지

## 7) 적용 범위

- 검색 결과 페이지
- 추천/목록/필터 결과가 비어있는 화면

## 8) 체크리스트

- [ ] 경고/에러 색상 사용하지 않았는가?
- [ ] 아이콘이 중립적인가?
- [ ] 문구가 간결한가?
- [ ] 버튼이 필요한 경우에만 노출되는가?
