# 디자인 시스템

Tasteam 프론트엔드의 디자인 시스템은 **일관된 UI/UX**, **접근성**, **개발 생산성**을 목표로 합니다.
이 문서는 디자인 시스템을 구축/운영하기 위해 필요한 기준을 정리합니다.

## 0) 구현 전략: shadcn/ui 기반

우리는 빠른 개발 속도를 위해 **shadcn/ui를 기본 베이스로 사용**합니다.
단, shadcn/ui는 "의존성으로 설치해서 쓰는 컴포넌트 라이브러리"라기보다, 레포 안으로 **컴포넌트 소스를 가져와(vendor) 우리 서비스에 맞게 커스텀**하는 방식에 가깝습니다.

원칙:

- 기본 컴포넌트는 shadcn/ui(보통 Radix UI 기반)를 우선 채택
- 서비스 톤앤매너/요구사항에 맞게 스타일/variant/API를 커스텀
- 커스텀 범위가 커질수록 "우리 컴포넌트"로 보고 운영(문서/변경 관리 포함)
- `features/`, `widgets/`, `pages/`에서는 **반드시 `src/shared/ui`만 사용**하며, shadcn 내부 구현은 외부로 노출하지 않음

운영 관점에서의 핵심 결정들은 `docs/디자인-시스템/결정사항.md`에서 관리합니다.

## 1) 목표와 범위

### 목표

- 화면/컴포넌트 간 시각적 일관성 유지
- 상태(hover/active/disabled/loading/error) 정의 표준화
- 접근성(키보드/스크린리더/명도 대비) 기본 충족
- 컴포넌트 재사용을 통한 개발 속도 향상
- 다크/라이트 등 테마 확장 가능한 구조 마련

### 범위(권장)

- Foundations(기초 규칙): 색상/타이포/간격/반경/그림자/레이어/모션
- Design tokens(코드로 관리되는 토큰)
- Shared UI 컴포넌트(재사용 가능한 프리미티브)
- 패턴/가이드(폼, 피드백, 네비게이션 등)
- 운영 프로세스(추가/변경/폐기, 버전, 리뷰)

## 2) 레포 구조(권장)

현재 레포의 기본 구조(FSD)를 유지하면서 아래처럼 배치합니다.

- 토큰/테마/전역 스타일: `src/shared/styles/`
  - 예: `src/shared/styles/global.css`, `src/shared/styles/theme.ts`
- 공용 UI 컴포넌트: `src/shared/ui/`
  - 예: `src/shared/ui/Button/`, `src/shared/ui/Text/`
- 도메인/시나리오 결합 UI는 상위 레이어로 이동
  - `features/`, `widgets/`, `pages/`에는 **조합/화면 구성** 중심으로 둡니다.

## 3) Design Tokens

디자인 토큰은 "디자인 의사결정"을 코드에서 재사용 가능하게 만든 값입니다.
**원시(primitives) → 의미(semantics) → 컴포넌트**로 계층을 나눕니다.

자세한 규칙은 `docs/디자인-시스템/토큰.md`를 참고합니다.

## 4) 컴포넌트 기준

공용 컴포넌트는 다음을 기본으로 갖춥니다.

- 변형(variant): `primary/secondary/ghost` 등 팀 합의된 최소 세트
- 크기(size): `sm/md/lg` 등(필요한 경우만)
- 상태(state): `default/hover/active/disabled/loading`
- 접근성: 키보드 포커스, `aria-*` 속성, 라벨링
- API 안정성: props 네이밍/기본값/변경 정책

자세한 체크리스트는 `docs/디자인-시스템/컴포넌트.md`를 참고합니다.

## 5) 접근성(필수)

디자인 시스템은 접근성 기준을 "선택"이 아니라 "기본값"으로 가져갑니다.

- 키보드만으로 모든 인터랙션 가능해야 함
- 포커스 스타일은 시각적으로 명확해야 함
- 폼 요소는 label/설명/에러 메시지를 스크린리더가 인식 가능해야 함
- 색상만으로 상태를 전달하지 않기(아이콘/텍스트/패턴 병행)

자세한 기준은 `docs/디자인-시스템/접근성.md`를 참고합니다.

## 6) 운영/변경 관리

- 새 컴포넌트/토큰 추가는 "요구사항 → 제안 → 리뷰 → 반영" 흐름을 따릅니다.
- Breaking change가 발생하면 명확한 마이그레이션 가이드를 함께 작성합니다.

자세한 내용은 `docs/디자인-시스템/운영-프로세스.md`를 참고합니다.
