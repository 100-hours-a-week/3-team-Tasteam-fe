# 컬러(Colors)

이 문서는 "브랜드 컬러를 중심으로 파생되는" 디자인 시스템 컬러 목록을 정리합니다.
목표는 색을 임의로 추가하지 않고, 정해진 역할(semantic)로만 UI를 구성하는 것입니다.

현재 전역 스타일에는 아래 브랜드 색상이 존재합니다.

- `--color-brand`: `#f97316`
- `--color-brand-strong`: `#ea580c`
- `--color-brand-soft`: `#fff1e6`

## 1) 컬러를 나누는 기준

- Palette(팔레트/스케일): 브랜드/중립색을 단계(예: 50~950)로 만든 "재료"
- Semantic(의미): 배경/텍스트/버튼/경고 등 UI 역할에 매핑된 "사용처"

원칙:

- 컴포넌트는 가능하면 semantic 컬러만 사용합니다.
- palette는 "의미 컬러를 만들기 위한 재료"로 제한합니다.

## 2) (권장) 팔레트 목록

브랜드 컬러를 중심으로 최소 아래 팔레트는 필요합니다.

- Brand(Primary) scale: `brand-50/100/200/300/400/500/600/700/800/900/950`
- Neutral(Gray) scale: `gray-50 ... gray-950`
- Success(Green) scale: `success-50 ... success-950`
- Warning(Amber/Yellow) scale: `warning-50 ... warning-950`
- Danger(Red) scale: `danger-50 ... danger-950`
- Info(Blue/Cyan) scale (선택): `info-50 ... info-950`

주의:

- 실제 UI는 대부분 neutral + brand + semantic 상태색(성공/경고/오류)만으로도 충분합니다.
- "보조/포인트 컬러"는 남발하면 시스템 일관성이 깨지므로, 필요가 확정될 때만 추가합니다.

## 3) (필수) Semantic 컬러 목록

shadcn/ui 기반으로 운영할 때, 실사용 관점에서 최소 아래 역할들이 필요합니다.

### Surface & Text

- `background`: 앱 최하위 배경
- `foreground`: 기본 텍스트
- `card`, `card-foreground`: 카드/패널 배경과 텍스트
- `popover`, `popover-foreground`: 팝오버/드롭다운 배경과 텍스트

### Brand / Emphasis

- `primary`, `primary-foreground`: 주요 CTA/강조(브랜드 중심)
- `secondary`, `secondary-foreground`: 보조 강조(중립 기반)
- `accent`, `accent-foreground`: 보조 포인트(상호작용/선택 강조)

### Muted / Subtle

- `muted`, `muted-foreground`: 약한 배경/보조 텍스트

### Status

- `destructive`, `destructive-foreground`: 위험/삭제/오류

추가(선택, 제품 요구가 명확할 때):

- `success`, `success-foreground`
- `warning`, `warning-foreground`
- `info`, `info-foreground`

### Borders / Focus / Inputs

- `border`: 구분선(디바이더)
- `input`: 입력 컴포넌트 보더/배경 기준(프로젝트 규칙에 맞게 정의)
- `ring`: 포커스 링/강조 테두리(접근성 핵심)

### Overlay

- `overlay`: 모달/드로어 배경 딤(권장)

### Data viz (선택)

- `chart-1` ~ `chart-n`: 차트 색상(서비스에서 필요할 때만)

## 4) 결정해야 하는 것(실무용)

아래를 확정하면 컬러 시스템이 "고정"됩니다.

- 라이트/다크 테마 여부(둘 다 지원할지)
- 브랜드 컬러를 `primary`로 사용할지, 또는 브랜드와 CTA를 분리할지
- `secondary/accent`를 실제로 사용할지(없으면 제거 가능)
- 상태색(success/warning/info)을 토큰으로 승격할지(실사용이 반복될 때)
- `ring` 색상을 브랜드로 할지, 접근성을 위해 별도 색으로 둘지

결정 기록은 `docs/디자인-시스템/결정사항.md`에서 관리합니다.
