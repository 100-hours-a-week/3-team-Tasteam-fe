# 아이콘 / 폰트

이 문서는 디자인 시스템에서 아이콘과 폰트를 어떻게 선택/운영할지 정리합니다.

## 1) 아이콘

### 선택(권장)

- 기본 아이콘 라이브러리: `lucide-react`
  - shadcn/ui 예제와 호환이 좋고, 라인 아이콘 스타일이 일관됩니다.

### 운영 원칙(권장)

- 기본 사이즈: 16 / 20 / 24 중 컴포넌트별로 표준화
- stroke/filled 혼용은 원칙적으로 금지(시각 일관성 저하)
- 아이콘 버튼은 `aria-label` 필수

### 추가 아이콘이 필요할 때

- 제품에서 "브랜드 아이콘"이 필요하면 `src/shared/assets/`에 SVG를 추가하고 `shared/ui/Icon` 같은 래퍼로 관리합니다.

## 2) 폰트

### 현재 상태

전역 기본 폰트는 `src/shared/styles/global.css`에서 설정되어 있습니다.

- 현재 설정: `Noto Sans KR`, system-ui fallback

### 선택지(권장 방향)

- 후보 A(유지): `Noto Sans KR`
  - 장점: 안정적, 한국어 가독성 우수, 별도 번들 관리 부담이 적음
- 후보 B(변경): `Pretendard`
  - 장점: 현대적인 인상, UI에서 자주 쓰이는 표준 폰트
  - 고려: 폰트 파일 크기/서브셋팅/로딩 전략을 정해야 함

### 운영 원칙(권장)

- 본문/제목의 기본 웨이트는 최소 세트(예: 400/500/600)로 제한
- 숫자 가독성이 중요한 화면은 숫자 탭(또는 숫자 폰트 정책)을 별도로 검토
- 폰트 변경은 UI 전체에 영향이 커서, 결정 후에는 큰 주기에서만 변경

결정 기록은 `docs/디자인-시스템/결정사항.md`에서 관리합니다.
