# 접근성(A11y) 기준

이 문서는 디자인 시스템 수준에서 지켜야 하는 접근성 기준을 정리합니다.

## 1) 키보드(필수)

- 모든 인터랙션은 키보드만으로 수행 가능해야 합니다.
- 포커스 이동 순서는 시각적/논리적 순서를 따릅니다.
- 포커스는 항상 "보이는" 스타일을 가져야 합니다(`:focus-visible` 권장).

## 2) 시맨틱/ARIA(필수)

- 가능한 경우 ARIA보다 올바른 HTML 시맨틱을 우선합니다.
  - 클릭 가능한 것은 `button`/`a` 사용
- 아이콘 버튼/닫기 버튼 등 텍스트가 없는 컨트롤은 `aria-label`을 제공합니다.
- 입력 요소는 `label` 또는 `aria-label`이 있어야 합니다.
- 도움말/에러 메시지는 `aria-describedby`로 입력과 연결합니다.

## 3) 폼(필수)

- 에러 상태는 색상만으로 표현하지 않습니다(텍스트/아이콘/메시지 병행).
- 에러 메시지는 사용자에게 "무엇을/어떻게" 고치면 되는지 알려야 합니다.

## 4) 색상 대비(권장: 기준 준수)

- 텍스트/배경 대비는 검증합니다(라이트/다크 모두).
- 상태 색상(성공/경고/오류)은 배경색/텍스트색 조합을 함께 고려합니다.

## 5) 모션(권장)

- 과도한 모션은 피하고, 가능한 경우 `prefers-reduced-motion`을 고려합니다.

## 6) 최소 체크리스트(컴포넌트 추가 시)

- Tab으로 접근 가능한가?
- `Enter`/`Space`로 활성화 가능한가(버튼/토글 등)?
- 포커스가 눈에 띄는가?
- 스크린리더로 컨트롤 목적을 이해할 수 있는가?
- 에러/로딩/비활성 상태가 명확한가?
