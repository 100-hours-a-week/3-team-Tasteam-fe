# 디자인 시스템 결정사항

이 문서는 디자인 시스템 구축/운영을 위해 "팀에서 반드시 결정해야 하는 것"을 모아둔 체크리스트입니다.
결정이 필요한 항목은 PR/회의를 통해 확정하고, 확정된 값은 이 문서에 기록합니다.

## 1) 기반 라이브러리: shadcn/ui

- 결정: shadcn/ui를 기본으로 사용하고, 컴포넌트 소스를 레포에 가져와 커스텀한다.
- 남은 결정:
  - 컴포넌트 배치 위치
    - 권장: `src/shared/ui/shadcn/` (가져온 원본/준원본)
    - 권장: `src/shared/ui/` (서비스 표준 컴포넌트; 래핑/확장한 최종 API)
  - 사용 규칙(Import 경계)
    - 결정: `features/`, `widgets/`, `pages/`에서는 **반드시 `src/shared/ui`만 사용**한다.
    - `src/shared/ui/shadcn`은 내부 구현 디테일이며, 외부 레이어에서 직접 import 하지 않는다.
  - 업데이트 전략
    - 권장: "추가(add) 시점의 스냅샷"으로 보고, 필요한 경우에만 수동으로 동기화

## 2) 컬러 토큰 포맷/네이밍

shadcn/ui 생태계는 보통 아래 semantic 토큰을 전제로 합니다.

- `--background`, `--foreground`
- `--card`, `--card-foreground`
- `--popover`, `--popover-foreground`
- `--primary`, `--primary-foreground`
- `--secondary`, `--secondary-foreground`
- `--muted`, `--muted-foreground`
- `--accent`, `--accent-foreground`
- `--destructive`, `--destructive-foreground`
- `--border`, `--input`, `--ring`

남은 결정:

- 색상 값을 `hsl(var(--token))` 방식으로 관리할지, `rgb(var(--token))` 방식으로 관리할지
  - 권장: shadcn 관례에 맞춰 HSL 기반(`hsl(var(--...))`)을 채택
- 테마 전환 방식
  - 후보 A: `.dark` 클래스 토글(shadcn 기본)
  - 후보 B: `[data-theme="dark"]` 속성 토글(프로덕트/SSR/서버 상태에 유리)

참고: 필요한 컬러 목록/역할 정의는 `docs/디자인-시스템/컬러.md`에 정리합니다.

## 3) 폰트(타이포그래피)

현재 전역 폰트는 `src/shared/styles/global.css`에서 설정합니다.

남은 결정:

- 기본 폰트 패밀리
  - 후보 A: `Noto Sans KR` (현재 적용)
  - 후보 B: `Pretendard` (자체 호스팅/번들 정책 필요)
- 숫자/영문 가독성 우선 여부(필요 시 영문 전용 폰트 병행)
- 기본 폰트 웨이트(400/500/600)와 제목/본문 스케일

권장 문서: `docs/디자인-시스템/아이콘-폰트.md`

## 4) 아이콘

남은 결정:

- 아이콘 라이브러리
  - 권장: `lucide-react` (shadcn/ui와 기본 궁합이 좋고, 아이콘 스타일이 일관됨)
- 사이즈 기준(예: 16/20/24)
- stroke/filled 정책(라인 아이콘 고정 vs 혼용 허용)

권장 문서: `docs/디자인-시스템/아이콘-폰트.md`

## 5) 컴포넌트 API/스타일링 원칙

남은 결정:

- 스타일 우선순위
  - 권장: Tailwind(유틸리티) 우선, 필요한 경우에만 Emotion 사용
- variant/size/state 기본 세트(버튼/입력/배지 등 핵심 컴포넌트부터)
- 접근성 최소 기준(포커스 링, aria, 키보드 동작)

관련 문서:

- `docs/디자인-시스템/컴포넌트.md`
- `docs/디자인-시스템/접근성.md`
