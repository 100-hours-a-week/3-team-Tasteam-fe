# Design Tokens

이 문서는 Tasteam 프론트엔드에서 디자인 토큰을 정의/사용하는 규칙을 정리합니다.
목표는 **디자인 값의 중복을 줄이고**, **테마 확장**과 **일관성 있는 UI**를 만드는 것입니다.

## 0) shadcn/ui와의 호환

우리는 shadcn/ui를 기본으로 사용하므로, 토큰은 shadcn 관례(semantic 컬러 토큰)를 최대한 따릅니다.
즉, 컴포넌트에서는 임의의 HEX/px 대신 토큰을 사용하고, 토큰은 테마(라이트/다크)에서 오버라이드됩니다.

필수 컬러 역할 목록은 `docs/디자인-시스템/컬러.md`를 참고합니다.

## 1) 토큰 계층(권장)

토큰은 아래 3단계로 관리합니다.

1. Primitives(원시값): 팔레트/기본 스케일
   - 예: `--gray-0`, `--gray-900`, `--blue-500`, `--space-4`
2. Semantics(의미값): UI 의미에 매핑된 값
   - 예: `--color-bg`, `--color-surface`, `--color-text`, `--color-border`
3. Component tokens(선택): 특정 컴포넌트의 디테일
   - 예: `--button-primary-bg`, `--input-border-focus`

원칙:

- 컴포넌트는 가능한 한 **semantic token**을 사용합니다.
- primitive 직접 사용은 제한적으로 허용(디자인적으로 "진짜 원시값"이 필요한 경우).

## 2) 네이밍 규칙(권장)

- 색상 primitive: `--<color>-<step>` (예: `--gray-900`, `--blue-500`)
- 의미 색상: `--color-<role>` (예: `--color-bg`, `--color-text-muted`)
- 간격: `--space-<n>` (예: `--space-1`, `--space-4`, `--space-8`)
- 반경: `--radius-<n>` (예: `--radius-2`, `--radius-6`)
- 그림자: `--shadow-<level>` (예: `--shadow-1`, `--shadow-2`)
- z-index: `--z-<layer>` (예: `--z-modal`, `--z-toast`)
- 모션: `--duration-<n>`, `--easing-<name>`

## 3) 테마(라이트/다크) 전략

테마 전환은 CSS 변수 오버라이드 방식이 가장 단순하고, Tailwind/Emotion과도 잘 결합됩니다.

- 기본: `:root`에 라이트 테마(또는 기본 테마) 정의
- 다크: `.dark` 또는 `[data-theme="dark"]`에서 semantic token을 오버라이드

예시(`src/shared/styles/global.css`):

```css
:root {
  /* primitives */
  --gray-0: 255 255 255;
  --gray-900: 17 17 17;

  /* semantics */
  --color-bg: var(--gray-0);
  --color-text: var(--gray-900);
}

.dark,
[data-theme='dark'] {
  --color-bg: var(--gray-900);
  --color-text: var(--gray-0);
}
```

주의:

- 의미 토큰만 테마에서 바뀌게 하면(primitive는 고정) 테마 관리가 쉬워집니다.
- 명도 대비(텍스트/배경)는 반드시 검증합니다.

## 4) Tailwind와의 연결(권장)

Tailwind 유틸리티를 쓰더라도 값은 토큰(CSS 변수)에 연결해 "디자인 값"을 한 곳에서 관리합니다.

예시 컨셉:

- `bg-[rgb(var(--color-bg))]`, `text-[rgb(var(--color-text))]`처럼 사용하거나
- Tailwind config에서 `colors`를 CSS 변수 기반으로 매핑합니다.

## 5) Emotion과의 연결(권장)

Emotion 사용 시에도 "값의 원천"은 토큰(CSS 변수)로 두고, 컴포넌트에서는 의미 중심으로 사용합니다.

예시 컨셉:

- `color: 'rgb(var(--color-text))'`
- 혹은 `theme.ts`에서 타입이 있는 래퍼를 제공(필요 시)

## 6) 토큰 변경/추가 체크리스트

- 새 값은 primitive보다 semantic으로 먼저 고려
- 신규 semantic 토큰은 "역할"이 명확해야 함(예: `--color-danger`는 OK, `--color-red-ish`는 NO)
- 라이트/다크 모두에서 값 정의(또는 폴백) 확인
- 적용 범위(어떤 컴포넌트에서 쓰는지) 문서에 기록
